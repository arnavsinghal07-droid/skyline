1. Create `supabase/migrations/003_waitlist.sql`:
   ```sql
   CREATE TABLE public.waitlist (
     id         UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
     email      TEXT        NOT NULL UNIQUE,
     created_at TIMESTAMPTZ NOT NULL DEFAULT now()
   );
   ALTER TABLE public.waitlist ENABLE ROW LEVEL SECURITY;
   -- No SELECT/UPDATE/DELETE policies — all access is via service role only
   -- Unauthenticated visitors cannot read the waitlist; admin client bypasses RLS
   ```
   - No RLS policies for `anon` or `authenticated` roles — the table is write-only via service role
   - UNIQUE constraint on `email` provides deduplication at the DB level

2. Run the migration against the local Supabase instance:
   ```bash
   npx supabase db push
   ```
   If `supabase` CLI is not available or errors, use `psql` directly or skip (the migration file itself is the deliverable — the user can apply it).

3. Create `src/app/api/waitlist/route.ts`:
   - Import `NextRequest` from `next/server`
   - Import `createAdminClient` from `@/lib/supabase/admin`
   - Import `sendWaitlistConfirmationEmail` from `@/lib/email`
   - Export `POST` handler:
     a. Parse body: `const { email } = await request.json()`
     b. Validate: if `!email` or `typeof email !== 'string'` or `!email.includes('@')`, return `Response.json({ error: 'Invalid email' }, { status: 400 })`
     c. Create admin client: `const admin = createAdminClient()`
     d. Insert: `const { error } = await admin.from('waitlist').insert({ email: email.toLowerCase().trim() })`
     e. If `error && error.code !== '23505'` (23505 = unique violation = already on list), return `Response.json({ error: 'Failed to save email' }, { status: 500 })`
     f. Fire-and-forget email: `await sendWaitlistConfirmationEmail(email)` — same async-but-no-throw pattern as sendWelcomeEmail
     g. Return `Response.json({ success: true })`
   - Do NOT use `<form>` or any form-related APIs (CLAUDE.md forbids HTML form tags)
   - Do NOT use the anon Supabase client — visitor is unauthenticated, RLS blocks anon writes

```typescript
export async function sendWaitlistConfirmationEmail(to: string) {
  if (!to || !process.env.RESEND_API_KEY) return

  const resend = new Resend(process.env.RESEND_API_KEY)

  try {
    await resend.emails.send({
      from: 'Sightline <onboarding@resend.dev>', // TODO: update from address when custom domain verified
      to,
      subject: "You're on the Sightline waitlist",
      html: `
        <div style="font-family: system-ui, sans-serif; max-width: 480px; margin: 0 auto;">
          <h2 style="color: #111;">You're on the list.</h2>
          <p style="color: #444; line-height: 1.6;">
            Thanks for your interest in Sightline. We'll be in touch when your spot opens up.
          </p>
          <p style="color: #444; line-height: 1.6;">
            Sightline is an AI-native product discovery platform — think Cursor, but for product managers.
          </p>
          <hr style="border: none; border-top: 1px solid #eee; margin: 24px 0;" />
          <p style="color: #999; font-size: 12px;">
            You received this because you joined the Sightline waitlist.
          </p>
        </div>
      `,
    })
  } catch (err) {
    console.error('[Sightline] Failed to send waitlist email:', err)
  }
}
```

- Log errors but do NOT throw — email failure should not block the 200 response to the user
- Inline HTML email — no react-email dependency (per RESEARCH.md)
