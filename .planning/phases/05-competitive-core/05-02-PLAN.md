API routes (created in 05-01):
```
GET  /api/competitors              → CompetitorRow[] with signal counts
POST /api/competitors              → { ...CompetitorRow } (201) or { error: 'competitor_limit_reached', limit } (402)
PATCH /api/competitors/[id]        → CompetitorRow
DELETE /api/competitors/[id]       → { success: true } (soft delete)
POST /api/competitors/[id]/scrape  → { jobId, status: 'pending' } (202)
POST /api/competitors/[id]/rescan  → { scanning: true, competitorId } (202)
POST /api/competitors/upload       → { signals_extracted, source } (multipart/form-data with file + competitor_id)
GET  /api/competitors/[id]/job-status → { status, signals_extracted, error_message, attempts }
```
