All routes follow the existing API pattern: import `createClient` from `@/lib/supabase/server`, call `supabase.auth.getUser()`, return 401 if no user, get `org_id` from users table. Use `NextResponse.json()` for responses.

**Route 1: `src/app/api/competitors/route.ts` — GET + POST**

GET: Fetch all competitors for the org where `deleted_at IS NULL`. Return JSON array of CompetitorRow objects. Include signal counts: for each competitor, also fetch count from `competitive_signals` table.

POST: Add a new competitor.
- Accept `{ name, g2_slug?, capterra_slug? }` in body.
- Enforce plan limit: count existing competitors (deleted_at IS NULL) for org, compare against `COMPETITOR_LIMITS[org.plan]`. If at limit, return 402 `{ error: 'competitor_limit_reached', limit }`.
- Get workspace_id from the workspaces table (first workspace for the org).
- Insert into competitors table.
- After insert, trigger auto-scan: use `unstable_after` from `next/server` to call `scanForCompetitorMentions()` for the new competitor. This runs after the response is sent.
- Return 201 with the created competitor row.

**Route 2: `src/app/api/competitors/[id]/route.ts` — PATCH + DELETE**

PATCH: Update competitor name, g2_slug, capterra_slug. Validate competitor belongs to user's org. Return updated row.

DELETE: Soft delete — set `deleted_at = new Date().toISOString()`. Do NOT hard delete. Return 200 `{ success: true }`.

**Route 3: `src/app/api/competitors/[id]/scrape/route.ts` — POST**

- Validate competitor belongs to org and has g2_slug or capterra_slug.
- Insert a `scraping_jobs` row with status 'pending'.
- Use `unstable_after` from `next/server` to run the scraping job asynchronously after response.
- The async function should:
  1. Update job status to 'processing'.
  2. Try scraping (G2 if g2_slug, Capterra if capterra_slug).
  3. For each scraped review, call `extractCompetitiveSignals()`.
  4. Upsert extracted signals into `competitive_signals` (use the UNIQUE constraint on competitor_id + review_source_id for dedup).
  5. Update job status to 'completed', set `signals_extracted` count.
  6. Update competitor's `last_scraped_at`.
  7. On failure: increment `attempts`, if attempts < 3 set status back to 'pending' (retry), if attempts >= 3 set status to 'failed' with error_message.
- Return 202 `{ jobId: job.id, status: 'pending' }`.

Note on `unstable_after`: Import as `import { unstable_after as after } from 'next/server'`. If the import fails at runtime (version compatibility), fall back to fire-and-forget fetch pattern: call an internal processing endpoint.

**Route 4: `src/app/api/competitors/[id]/rescan/route.ts` — POST**

- Validate competitor belongs to org.
- Use `unstable_after` to run `scanForCompetitorMentions()` after response.
- Return 202 `{ scanning: true, competitorId }`.

**Route 5: `src/app/api/competitors/upload/route.ts` — POST**

- Accept `multipart/form-data` with `file` (CSV) and `competitor_id` fields.
- Read the CSV file text content.
- Call `parseCsvReviews()` to parse and normalize.
- If source detection returns 'unknown', return 400 with error message.
- Get workspace_id from workspaces table.
- For each parsed review, call `extractCompetitiveSignals()`.
- Upsert extracted signals into `competitive_signals` table.
- Return 200 `{ signals_extracted: count, source: detectedSource }`.

Note: CSV upload is synchronous — the file is small enough (typically < 500 reviews). The extraction calls to Haiku are the bottleneck but acceptable for CSV upload UX.

**Route 6: `src/app/api/competitors/[id]/job-status/route.ts` — GET**

- Fetch latest scraping_job for competitor_id, ordered by created_at desc.
- Return `{ status, signals_extracted, error_message, attempts }`.
- If no job exists, return `{ status: 'none' }`.
